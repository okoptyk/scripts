# Дизайн: страница «Повторные продажи»

Дата: 2026-06-15
Проект: scripts-website (OkOptyk)

## Цель

Добавить на сайт новый этап «Повторные продажи» — пост-продажный цикл работы
с клиентом (сервисный звонок через 30 дней, кросс-продажа через 3–6 мес,
плановый осмотр через 1–1.5 года), на трёх языках (RU/UA/PL).

## Решения (согласованы с пользователем)

1. **«Футер»** = добавить ссылку «Повторные продажи» в верхнее меню (навбар)
   на всех страницах. Настоящего блока-футера на сайте нет; не создаём.
2. **Место в воронке** = последний этап цикла, после «Готовности заказа».
3. **Структура страницы** = 3 вкладки по этапам (1/2/3) + лайфхаки +
   шаблон для контактных линз.
4. **Цепочка `stage-path`** обновляется только на «Готовности заказа»
   (кнопка теперь ведёт на новый этап) и на самой новой странице.
   Остальные страницы в части `stage-path` не трогаем.

## Новый файл

`povtornye-prodazhi.html`, `data-page="povtor"` (имя латиницей — без ловушки
смешанной раскладки, как у `kvалификация.html`).

Структура (по образцу `gotovnost-zakaza.html`):

- Навбар (новый пункт «Повторные продажи» с классом `active`).
- `page-header` — заголовок «Повторные продажи» + подзаголовок
  «Реализуем потенциал повторной продажи: забота о клиенте после покупки».
- `meta-block`:
  - ЦКП: «Максимально реализован потенциал повторной продажи».
  - Стартовая точка: «По клиенту успешно завершена сделка; создана новая
    заявка в воронке повторной продажи».
  - Дедлайн: «30 / 60 / 90 / 120 / 150 / 180 / 180+ дней после продажи».
  - Ответственный: «Продавец-консультант, Менеджер».
- `info-box` — короткое вступление о сути этапа.
- **3 вкладки этапов** (`switchTab`):
  - «📅 30 дней · Сервисный звонок»
  - «📅 3–6 мес · Вторая пара»
  - «📅 1–1.5 года · Плановый осмотр»
  - В каждой вкладке: `phase-goal` (цель), диалог `.msg.consultant`
    (со сценариями А/Б, где есть), и внутри — переключатель языка скриптов
    `.lang-btn` (RU/UA/PL) с обёртками `id="povtor1-ru/ua/pl"` и т.д.,
    чтобы copy.js определял язык реплик.
- **`tip-block`** — лайфхаки (CRM, «дари пользу, а не продавай»).
- **Шаблон для контактных линз** — `template-box` + ручная кнопка
  `copyText` (сообщение в мессенджер за 10 дней до окончания линз),
  на трёх языках через свой `.lang-btn`.
- `next-stage` — `stage-path` со всеми этапами + «Повторные продажи»
  как финальным; «🎉 Вы прошли весь цикл!» + кнопка «← На главную».

**Копирование:** `body data-copy="all"` → авто-кнопки copy.js на всех
`.msg.consultant`; шаблон линз — ручной `copyText` (как на gotovnost).

## Изменения в существующих файлах

1. **Навбар на всех 7 существующих HTML** — добавить `<li>` со ссылкой
   `povtornye-prodazhi.html` «Повторные продажи» после «Готовность заказа».
2. **index.html** — новая карточка `.nav-card` «Повторные продажи»

3. **gotovnost-zakaza.html** — `next-stage`: вместо «🎉 прошли все этапы»

   продажи →».
4. **js/i18n.js**:
   - `nav.ua` / `nav.pl` — добавить `povtornye-prodazhi.html`.
   - `titles` — добавить `povtor` (ua/pl).
   - `pages.index.cards` (ua/pl) — добавить новую карточку.
   - `pages.gotov.next` (ua/pl) — обновить под новую цепочку и кнопку.
   - `pages.povtor` (ua/pl) — новая секция со всеми ключами `data-i18n`.

## Соглашения
